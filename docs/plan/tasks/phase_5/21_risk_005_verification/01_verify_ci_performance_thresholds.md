# Task: Verify CI Performance Thresholds (Sub-Epic: 21_Risk 005 Verification)

## Covered Requirements
- [AC-RISK-005-03], [AC-RISK-005-04]

## Dependencies
- depends_on: [none]
- shared_components: [./do Entrypoint Script]

## 1. Initial Test Written
- [ ] Create a Python script `tests/test_ci_performance.py` that parses `target/presubmit_timings.jsonl` and `target/traceability.json` to verify that all requirements related to timing are satisfied.
- [ ] Write a test case that simulates a clean checkout (empty `target/` except for essential artifacts) and asserts that `./do presubmit` completes within the 15-minute (900s) budget on the CI runner environment.

## 2. Task Implementation
- [ ] Implement a check in `.tools/ci.py` or as a standalone script that:
    - Reads `target/presubmit_timings.jsonl` generated by `./do presubmit`.
    - Asserts the total duration (sum of all steps) is <= 900,000 ms.
    - Exits non-zero if the budget is exceeded, ensuring CI failure.
- [ ] Configure GitLab CI pipeline (`.gitlab-ci.yml`) to set a hard `timeout: 25m` for `presubmit-linux`, `presubmit-macos`, and `presubmit-windows` jobs to enforce `[AC-RISK-005-03]`.
- [ ] Add a step in the CI pipeline to print a summary of `target/presubmit_timings.jsonl` at the end of each job.

## 3. Code Review
- [ ] Verify that the 15-minute limit for clean checkouts is consistently met on the `presubmit-linux` runner.
- [ ] Ensure that the CI timeout configuration is correctly applied to all three platform jobs.

## 4. Run Automated Tests to Verify
- [ ] Run `python3 tests/test_ci_performance.py` in the CI environment (or simulate it).
- [ ] Execute `./do presubmit` on a clean checkout and verify it passes within 15 minutes.

## 5. Update Documentation
- [ ] Update `target/presubmit_timings.jsonl` schema documentation in `8_risks_mitigation.md` if any changes were made to the implementation.

## 6. Automated Verification
- [ ] Check CI pipeline logs to confirm all `presubmit-*` jobs finished within 25 minutes.
- [ ] Verify `target/traceability.json` shows `[AC-RISK-005-03]` and `[AC-RISK-005-04]` as covered.

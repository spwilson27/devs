# Task: 01_Automated Risk State Transitions (Sub-Epic: 08_Risk 006 Verification)

## Covered Requirements
- [RISK-BR-006]

## Dependencies
- depends_on: [none]
- shared_components: [Traceability & Verification Infrastructure]

## 1. Initial Test Written
- [ ] Create a Python unit test in `.tools/tests/test_risk_transition.py` that mocks the `target/traceability.json` report and a risk document `docs/plan/requirements/8_risks_mitigation.md`.
- [ ] The test must verify that when a requirement of type `Risk` (tagged `RISK-[0-9]{3}`) has status `Mitigated` but the `covering_tests` in the report show a failure (non-zero exit code or `failed` status), the transition logic identifies this as a regression.
- [ ] Verify that the tool can update the status of the risk from `Mitigated` back to `Open` in the markdown file automatically.

## 2. Task Implementation
- [ ] Enhance `.tools/verify_requirements.py` (or create a dedicated `risk_manager.py`) to parse the risk status from `8_risks_mitigation.md`.
- [ ] Implement logic to read the authoritative `target/traceability.json` (generated by `./do test`).
- [ ] For each risk in `Mitigated` state, check the status of its associated `covering_tests`.
- [ ] If any covering test failed, perform a regex-based replacement in `8_risks_mitigation.md` to set the status to `Open`.
- [ ] Ensure the update is atomic and maintains the formatting of the document.

## 3. Code Review
- [ ] Verify that the regex for status transition is robust and doesn't affect non-risk sections.
- [ ] Ensure the tool provides clear logs (stdout) when a risk is transitioned back to `Open`.
- [ ] Verify that `target/traceability.json` is correctly treated as the authoritative source as per [RISK-BR-006].

## 4. Run Automated Tests to Verify
- [ ] Run `pytest .tools/tests/test_risk_transition.py` and ensure it passes.
- [ ] Manually trigger a test failure for a mitigated risk and run the tool to verify the markdown update.

## 5. Update Documentation
- [ ] Update `docs/plan/summaries/8_risks_mitigation.md` to reflect that the automated transition is now implemented and active in the pipeline.

## 6. Automated Verification
- [ ] Run `.tools/verify_requirements.py --verify-doc docs/plan/requirements/8_risks_mitigation.md target/traceability.json` (or the equivalent new command) and verify it correctly identifies and reports the regression.

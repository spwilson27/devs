# Task: Configure CI Artifacts for Presubmit Timings (Sub-Epic: 09_Risk 011 Verification)

## Covered Requirements
- [RISK-BR-015]

## Dependencies
- depends_on: [none]
- shared_components: [./do Entrypoint Script]

## 1. Initial Test Written
- [ ] Create a Python test `tests/test_ci_artifact_config.py` that:
    - Loads `.gitlab-ci.yml` using a YAML parser.
    - Verifies that the `artifacts` section exists for all relevant presubmit jobs (e.g., `presubmit-linux`, `presubmit-macos`, `presubmit-windows`).
    - Verifies that `target/presubmit_timings.jsonl` is included in the `paths` list.
    - Verifies that `expire_in` is set to `7 days`.
    - Verifies that `when` is set to `always`.

## 2. Task Implementation
- [ ] Update `.gitlab-ci.yml` at the repository root.
- [ ] Add `target/presubmit_timings.jsonl` to the `artifacts` configuration in the `presubmit` template or individual jobs.
- [ ] Ensure the configuration looks like:
    ```yaml
    artifacts:
      paths:
        - target/coverage/report.json
        - target/presubmit_timings.jsonl
        - target/traceability.json
      expire_in: 7 days
      when: always
    ```
- [ ] Verify that the file `target/presubmit_timings.jsonl` is actually generated by `./do presubmit`.

## 3. Code Review
- [ ] Ensure the YAML indentation is correct.
- [ ] Verify that the `when: always` setting correctly ensures artifacts are saved even on failure.

## 4. Run Automated Tests to Verify

- [ ] Run `./do lint` and ensure `yamllint` (if available) passes on the new `.gitlab-ci.yml`.

## 5. Update Documentation

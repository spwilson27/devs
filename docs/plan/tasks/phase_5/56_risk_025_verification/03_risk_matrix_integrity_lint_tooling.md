## 2. Task Implementation

- [ ] In `.tools/verify_requirements.py`, add or extend the `check_risk_matrix_integrity()` function (must be callable independently from the full traceability check) with the following sub-checks:

  **Sub-check 1 — Parse risk matrix table:**
  - Parse `docs/plan/specs/8_risks_mitigation.md` (or `docs/plan/requirements/8_risks_mitigation.md` as canonical source) for the §1 matrix table.
  - Extract all rows matching the pattern `| [RISK-NNN] |` using a regex such as `r'\|\s*\[RISK-(\d+)\]'`.
  - Build a dict: `risk_map: Dict[str, RiskEntry]` where `RiskEntry` has fields `severity_score: int`, `mitigation_id: str`, `status: str`.

  **Sub-check 2 — Parse mitigation sections:**
  - Scan `docs/plan/specs/8_risks_mitigation.md` for mitigation section headers matching `[MIT-NNN]` using regex `r'\[MIT-(\d+)\]'`.
  - Build a set: `mit_ids: Set[str]`.

  **Sub-check 3 — Cross-reference RISK ↔ MIT:**
  - For every `risk_id` in `risk_map`, check that `risk_map[risk_id].mitigation_id` is in `mit_ids`.
  - For every `mit_id` in `mit_ids`, check that a corresponding `RISK-<N>` entry exists in `risk_map` (matching by numeric suffix: `MIT-025` ↔ `RISK-025`).
  - Collect violations into: `unmitigated_risks: List[str]` and `orphaned_mitigations: List[str]`.

  **Sub-check 4 — High-severity covering test check:**
  - For every risk in `risk_map` where `severity_score >= 6` and `status != "Accepted"`:
    - Run `grep -r "// Covers: RISK-<N>" crates/` (or use Python's `pathlib` + `re` to scan Rust files).
    - If no annotation found, append to `uncovered_high_severity: List[str]`.

  **Sub-check 5 — Write violations to traceability.json:**
  - Populate `traceability_data["risk_matrix_violations"]` as a list of dicts:
    ```json
    [
      {"type": "unmitigated_risk", "id": "RISK-NNN", "detail": "No matching MIT-NNN section"},
      {"type": "orphaned_mitigation", "id": "MIT-NNN", "detail": "No matching RISK-NNN matrix row"},
      {"type": "uncovered_high_severity", "id": "RISK-NNN", "detail": "No // Covers annotation"}
    ]
    ```
  - If any violations exist, exit with non-zero status code after writing the JSON.

- [ ] In `.tools/lint_risk_matrix.py` (create as a standalone script), implement `main()` that:

  3. Exits `0` if `risk_matrix_violations` is empty, `1` otherwise.

## 1. Initial Test Written
- [ ] Create a shell script `scripts/verify_isolation_stress.sh` that:
    - Runs a set of existing E2E tests 100 times in a loop.
    - Specifically, use `cargo test --test '*' -- --test-threads 1`.
    - Asserts that all 100 runs exit with status 0.
    - Greps the output for any occurrence of `EADDRINUSE` or `address already in use`.

## 2. Task Implementation
- [ ] Implement the `scripts/verify_isolation_stress.sh` script.
    - Add logic to capture and summarize the output of each of the 100 runs.
    - Ensure it cleans up any potential leftover `TempDir` artifacts (though `ServerHandle` should handle this).
    - Add a check for "successful port binding" logs from the server for each of the 100 starts.
- [ ] Integrate this stress test into the Phase 5 verification phase (or as a standalone task for this sub-epic).
- [ ] Execute the script and ensure it completes 100 iterations without failure.

## 3. Code Review
- [ ] Confirm that 100 runs is exactly the number of iterations performed.
- [ ] Verify that the script correctly detects `EADDRINUSE` even if the test failure doesn't immediately cause a non-zero exit (e.g., if a sub-task fails but is swallowed).
- [ ] Ensure that `test-threads = 1` is strictly enforced during this stress test.

## 4. Run Automated Tests to Verify
- [ ] Run the `scripts/verify_isolation_stress.sh` and ensure it passes all 100 iterations.

## 5. Update Documentation
- [ ] Update `docs/plan/verification/phase_5_results.md` (or equivalent) to record the successful stress-verification of E2E isolation.

## 6. Automated Verification
- [ ] Run the script and check its final exit code and the presence of "100 runs successful" in its output.
- [ ] Verify that no `EADDRINUSE` errors are found in the logs generated by the script.

## 2. Task Implementation

- [ ] Open `./do` and locate the `coverage` subcommand implementation.
- [ ] Refactor `./do coverage` to run **two separate** `cargo llvm-cov` invocations:
  ```sh
  # Step 1: Unit test coverage (QG-001, QG-002)
  cargo llvm-cov \
      --workspace \
      --lcov --output-path target/coverage/unit.lcov.info \
      -- --test-threads=4

  # Step 2: E2E test coverage — only tests matching *_e2e* (QG-003, QG-004, QG-005)
  # Covers: MIT-023, AC-RISK-023-02
  LLVM_PROFILE_FILE="/tmp/devs-coverage-%p.profraw" \
  cargo llvm-cov \
      --workspace \
      --lcov --output-path target/coverage/e2e.lcov.info \
      --test '*_e2e*' \
      -- --test-threads=1
  ```
- [ ] Add a `--dry-run` flag to `./do coverage` that prints the two commands above without executing them (used by the Python test harness in §1).
- [ ] After both invocations, run the gate-check script (`./do coverage` already calls `.tools/ci.py` or similar) separately for:
  - `unit.lcov.info` → QG-001 (≥90% unit line coverage).
  - `e2e.lcov.info` → QG-003 (CLI ≥50%), QG-004 (TUI ≥50%), QG-005 (MCP ≥50%), QG-002 (aggregate E2E ≥80%).
- [ ] Ensure the gate-check script does NOT combine `unit.lcov.info` and `e2e.lcov.info` into a single file before computing QG-003/004/005.
- [ ] Add comments to `./do` above the E2E `cargo llvm-cov` invocation referencing `AC-RISK-023-02` and `RISK-023-BR-004`.
- [ ] Confirm that `cargo llvm-cov ... --test '*_e2e*'` correctly filters — run with `--no-report` first and verify that only test binaries matching the `*_e2e*` pattern are listed.

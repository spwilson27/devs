# Sub-Epic: 18_Risk_004_Verification

## Overview
This sub-epic verifies the mitigation for **RISK-004**: Agent adapter CLI interface breakage from upstream changes. The implementation ensures that:
1. Adapter CLI versions are tracked and freshness-enforced via `target/adapter-versions.json`.
2. Rate-limit detection uses case-insensitive substring matching (no regex dependency).
3. Missing adapter binaries are detected pre-spawn with immediate failure (no retry).
4. All 5 adapter compatibility tests pass against captured versions.

## Covered Requirements
All requirements in this sub-epic are traced to the parent risk mitigation specification (docs/plan/specs/8_risks_mitigation.md):

| Requirement ID | Type | Description |
|---|---|---|
| [RISK-004-BR-003] | Functional | `target/adapter-versions.json` MUST be regenerated by `./do setup` and its `captured_at` timestamp MUST be within 7 days; `./do lint` MUST fail if the file is absent or stale |
| [RISK-004-BR-004] | Functional | Rate-limit pattern matching MUST use case-insensitive substring search against the full stderr content; regex-based matching is not required and MUST NOT be introduced |
| [MIT-004] | Technical | Mitigation for RISK-004: adapter CLI interface breakage from upstream changes |
| [AC-RISK-004-01] | Acceptance | Each of the 5 adapter compatibility tests passes with the version captured in `target/adapter-versions.json` |
| [AC-RISK-004-02] | Acceptance | A stage targeting a missing binary transitions to `Failed` with `failure_reason: "binary_not_found"` within 100ms and does NOT retry |

## Shared Components

### ./do Entrypoint Script (Owner)
This sub-epic **owns** the adapter version tracking logic in the `./do` script:
- `./do setup` probes all 5 adapters and generates `target/adapter-versions.json` with ISO8601 timestamp
- `./do lint` validates the file exists and `captured_at` is within 7 days
- Stale or missing compatibility check treated as lint failure

### devs-adapters (Owner)
This sub-epic **owns** the rate-limit detection and binary presence validation in `devs-adapters`:
- Case-insensitive substring matching for rate-limit patterns (`.to_lowercase().contains()`)
- No `regex` crate dependency (enforced by lint check)
- Pre-spawn binary existence check using `which` crate or `std::fs::metadata`
- Immediate failure with `failure_reason: "binary_not_found"` (non-retryable)
- Adapter CLI flags defined as `pub const &str` constants in each adapter's `config.rs`

### devs-scheduler (Consumer)
This sub-epic **consumes** from `devs-scheduler`:
- Retry logic that treats `binary_not_found` as non-retryable per [3_PRD-BR-021]
- Stage failure handling with proper `failure_reason` propagation

## Verification Strategy

### Unit Tests
- Rate-limit detection with zero exit code returns `false` (Task 02)
- Rate-limit detection with non-zero exit and matching patterns (Task 02)
- Case-insensitive substring matching verification (Task 02)

### Integration Tests
- Adapter version capture and validation (Task 01, Task 03)
- Missing binary spawn failure within 100ms (Task 02)
- No retry on `binary_not_found` (Task 02)

### Lint Checks
- `./do lint` validates `target/adapter-versions.json` freshness (Task 01)
- Adapter flag literal scan ensures `const &str` usage (Task 03)
- `regex` crate absence in `devs-adapters/Cargo.toml` (Task 02)

### Traceability
All tests must include `// Covers: REQ-ID` annotations mapping to the requirements listed above. The `./do test` command generates `target/traceability.json` which must show 100% coverage for all five requirements.

## Success Criteria
- [ ] All five requirements have passing tests with traceability annotations
- [ ] `./do test` exits 0 with `target/traceability.json` showing full coverage
- [ ] `./do lint` passes including the adapter version freshness check
- [ ] `./do setup` generates valid `target/adapter-versions.json` with all 5 adapters
- [ ] Manual verification: rate-limit detection never triggers on zero exit code
- [ ] Manual verification: missing binary fails within 100ms without retry
- [ ] Manual verification: `regex` crate NOT present in `devs-adapters/Cargo.toml`
- [ ] All 5 adapter compatibility tests pass against captured versions

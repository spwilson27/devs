# Task: Add Tailwind shadow utilities mapped to design tokens (Sub-Epic: 42_Shadow_Task_Cards)

## Covered Requirements
- [7_UI_UX_DESIGN-REQ-UI-DES-047-3], [7_UI_UX_DESIGN-REQ-UI-DES-047-3-1], [7_UI_UX_DESIGN-REQ-UI-DES-047-3-2]

## 1. Initial Test Written
- [ ] Create a unit/integration test at `packages/ui/src/__tests__/tailwind-shadow-config.test.ts` that imports the project's `tailwind.config.(js|cjs|ts)` and asserts that `theme.extend.boxShadow` contains two entries (e.g., `devs-sm` and `devs-md`) whose values resolve to the shadow token strings from `packages/ui/src/tokens/shadows.ts`.

## 2. Task Implementation
- [ ] Modify `tailwind.config.*` to add custom boxShadow utilities that reference the shadow tokens created in task 01. Implementation options:
  - Import the token module in the config (if using JS/TS) and set `theme.extend.boxShadow['devs-sm'] = SHADOWS.sm`, `['devs-md'] = SHADOWS.md`; OR
  - Use CSS custom properties generated by `packages/ui/dist/shadows.css` and reference them in a small `input.css` used for verification: `box-shadow: var(--devs-shadow-sm);`.
  - Ensure utilities follow the namespace `devs-` to avoid colliding with Tailwind defaults.

## 3. Code Review
- [ ] Verify the Tailwind config change does not remove default Tailwind shadows, uses a namespaced key (`devs-sm`/`devs-md`), and that the test imports the same token source (no duplicated literal values). Confirm the config works when building the UI bundle (no runtime errors from requiring the token module).

## 4. Run Automated Tests to Verify
- [ ] Run the UI package test suite and, as an additional verification, run a quick Tailwind build step to generate CSS from a minimal entrypoint that uses the new utilities and grep the output to confirm the box-shadow values are present. Example command sequence to document/run in CI:
  - `node packages/ui/scripts/generate-css-tokens.js`
  - `npx tailwindcss -i packages/ui/src/styles/input.css -o packages/ui/dist/output.css --content "packages/ui/src/**/*.{ts,tsx,js,jsx}"`
  - `grep -E "devs-sm|devs-md" packages/ui/dist/output.css`

## 5. Update Documentation
- [ ] Update `docs/ui/design_tokens.md` and the UI README with instructions showing how to use `devs-sm` and `devs-md` in component classNames and via CSS variables; include the requirement IDs.

## 6. Automated Verification
- [ ] Add a CI step `verify:tailwind-shadows` that runs the generate + tailwind build + grep sequence and fails if expected values are not present. Document how to run locally.
